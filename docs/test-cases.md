# Test Cases — Fake Store API

> This document follows the IEEE 829 / ISTQB test case structure.
> Each test case is documented before implementation.

## Template

| Field        | Description                                  |
|--------------|----------------------------------------------|
| **ID**       | TC-XXX                                       |
| **Title**    | Short, behavior-describing title             |
| **Resource** | Which API resource (Products, Carts, Users)  |
| **Method**   | HTTP method (GET, POST, PUT, DELETE)         |
| **Endpoint** | Full path                                    |
| **Preconditions** | What must be true before the test runs  |
| **Steps**    | Numbered steps                               |
| **Expected** | What the response should contain             |
| **Actual**   | Filled after execution                       |
| **Status**   | PASS / FAIL / BLOCKED                        |
